# Dispatcher v0.2.0 Codex Handoff Router

Draft release date: 2026-06-17

This draft prepares the first Dispatcher 2.0 publication as the new GitHub
project [Swaggyllz/dispatcher-codex-handoff](https://github.com/Swaggyllz/dispatcher-codex-handoff).
It should not be pushed over the existing Dispatcher 1.0 repository.

## Summary

Dispatcher 2.0 introduces a Codex-first handoff workflow for quota pressure.
When the native Codex `/v1/responses` route observes a 429, `retry-after`, or
reliable quota signal, Dispatcher records observable telemetry and, for
emergency conditions, creates a `dispatcher_handoff.v1` package. The dashboard
shows the latest handoff and lets the user explicitly continue through
`provider-auto` as degraded execution.

## Included

- `dispatcher_handoff.v1` emergency handoff package schema.
- Native Codex quota/rate-limit telemetry persisted in SQLite.
- Header-derived normalized quota headroom when reliable limit/remaining pairs are present.
- Dashboard visibility for the latest quota signal and emergency handoff.
- Copyable continuation prompt from the handoff card.
- User-approved fallback continuation through `provider-auto`.
- Telemetry for the latest non-streaming fallback continuation.
- Copyable primary-route review prompt for comparing fallback output before returning to Codex.
- App-like simple dashboard as the default UI, with the detailed operator dashboard behind Professional mode.
- Release metadata, CLI help, package versions, and web document metadata aligned to `0.2.0` / Codex Handoff Router.
- Updated project manual and 2.0 release strategy.

## Not Included

- Claude Code-specific handoff behavior.
- Planned 10% handoff trigger.
- Automatic background fallback execution.
- Hosted Responses tool emulation for third-party providers.
- Hidden reasoning or private context migration.
- Claims that fallback models are equivalent to Codex-native execution.
- Streaming fallback continuation persistence.
