# Dispatcher v0.3.0 Fallback Worker Certification

Release date: unreleased local preparation

Dispatcher v0.3.0 is a Dispatcher 2.0 milestone for Codex Handoff Router. It is
not Dispatcher 3.0 and not a generic model switcher.

## Summary

v0.3.0 adds a fallback worker certification layer. Tagged handoff continuation
requests are eligible only for model-level certified fallback workers. Ordinary
`provider-auto` routing remains available for non-handoff traffic.

## Included

- Model-level `handoff_certification` schema on provider models.
- Built-in fallback worker eval fixtures for:
  - `handoff_text_only`
  - `handoff_code_patch`
  - `handoff_tool_capable`
  - `handoff_long_context`
- Handoff-only eligibility filtering for requests tagged with
  `handoff_package_id`.
- `/v1/providers` visibility for model-level certification profiles.
- Handoff continuation telemetry fields:
  - `certification_labels`
  - `eligibility_reason`
- Dashboard visibility for certified worker count and saved continuation
  certification state.
- Local demo provider text-only certification for local testing only.

## Not Included

- Publishing to the Dispatcher 1.0 `origin` repository.
- Claude Code-first handoff behavior.
- Default silent model switching.
- Hosted Responses tool emulation for third-party providers.
- Claims that fallback workers are equivalent to native Codex.
- Production certification for domestic models without operator eval evidence.
