# Dispatcher v0.1.0-alpha.1

Release date: 2026-06-15

This is the first public alpha of Dispatcher, an OpenAI-compatible local
routing service for selecting and failing over between LLM providers.

## Included

- OpenAI-compatible chat completions, responses, and model discovery routes.
- Provider health checks, circuit breaking, retry handling, and routing rules.
- Codex provider-auto support, including function tools and streamed tool calls.
- A static React dashboard served by the Dispatcher process.
- Linux x86_64, macOS Apple Silicon, and Windows x86_64 release archives.

## Alpha limitations

- Release binaries are unsigned.
- Configuration is file- and environment-variable-based.
- The HTTP service does not provide multi-user authentication or tenant
  isolation. Bind it to a trusted interface unless a secure reverse proxy is
  used.
- Provider compatibility varies because upstream APIs differ.

## Upgrade and rollback

Back up `dispatcher.toml` and the runtime database before upgrading. To roll
back, stop Dispatcher and restore the previous binary, dashboard directory,
configuration, and database backup.

## Verification

Release candidates must pass:

```bash
./scripts/check-open-source-readiness.sh
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
pnpm --dir web format:check
pnpm --dir web typecheck
pnpm --dir web build
```
