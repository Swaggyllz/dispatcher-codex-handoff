# Support

Before opening an issue:

1. Read the [README](README.md) and [MVP manual](docs/mvp-user-manual-zh.md).
2. Reproduce with the Demo provider when possible.
3. Run `GET /v1/health`, `GET /v1/providers`, and inspect non-sensitive service logs.
4. Search existing issues.

Bug reports should include the operating system, Dispatcher version or commit, client,
request mode (`auto` or `provider-auto`), sanitized error text, and reproduction steps.

Never post API keys, bearer tokens, `ChatGPT-Account-Id`, full request headers, private
source code, or runtime databases.
